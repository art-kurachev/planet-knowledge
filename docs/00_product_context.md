# Product Context

## Продукт

**Planet CRM** — внутренняя веб-CRM/ERP система для строительной компании (жилые комплексы, Москва).

Это рабочая система контроля процессов, а не интерфейс для вдохновения. Используется ежедневно как операционная среда 8+ часов в день.

## Модули

- **Закупки и снабжение** — запросы, счета, сравнение поставщиков
- **Финансовые процессы** — согласование, оплата
- **Договорная работа**
- **Управление задачами** — канбан, статусы, дедлайны
- **Отчетность** — для руководства
- **CMS** — управление сайтами
- **CRM** — работа с клиентами, включая недвижимость

## Платформа

- Web-only
- Светлая тема
- Синяя визуальная айдентика
- Интерфейс насыщенный, но структурный

## Среда использования

Это не стартап-дашборд. Реальный контекст работы:

- Открыто 6–15 вкладок одновременно
- Параллельно WhatsApp / Telegram / звонки
- Частые прерывания
- Работа кусками по 5–20 минут
- Возвраты в интерфейс после отвлечений

**Что это значит для UX:**
- Интерфейс должен быть устойчив к прерыванию
- Состояния должны быть очевидны без "вспоминания"
- Нельзя требовать удержания сложного контекста в голове

## Ритм системы

Planet — не "творческая среда". Это среда подтверждений, проверок, согласований и фиксации статусов.

UI должен быть: строгим, спокойным, без "игры", без эмоциональных перегибов.

## Ошибки и ответственность

- Ошибка = деньги
- Ошибка = срыв сроков
- Ошибка = конфликт с руководством

Поэтому:
- Нельзя маскировать критические действия
- Нельзя делать неоднозначные кнопки
- Нельзя ставить destructive-действия рядом с подтверждающими
- Подтверждение должно быть осмысленным, а не декоративным

## Принцип нагрузки

Интерфейс может быть нагруженным, но не хаотичным. Есть разница между информационной плотностью и визуальным шумом.

**Система может быть насыщенной данными, но структура всегда должна быть очевидной.**

## Визуальные принципы

- Светлый интерфейс
- Основной цвет — синий (контрастный, рабочий)
- Белые блоки как основные контейнеры
- UI: контрастный, читаемый, структурный, без декоративности

## Приоритеты развития

1. Снижение времени принятия решения
2. Повышение читаемости статусов
3. Уменьшение когнитивной нагрузки
4. Предсказуемость навигации
5. Контроль и прозрачность
