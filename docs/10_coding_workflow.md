# Coding Workflow — процесс верстки

## Главный принцип

Вёрстка 1 в 1 с Figma. Pixel-perfect. Никаких допущений и приближений.

---

## Процесс

### 1. Уточнение перед стартом

Перед началом работы агент **обязан** задать вопросы, если:
- Неясен scope — что именно верстать
- Не определено поведение (hover, active, disabled)
- Есть адаптив или только десктоп
- Непонятна структура данных (API или мок)
- Есть конфликт между макетом и существующими правилами

**Не начинать работу, пока не прояснены все неоднозначности.**

### 2. Чтение макета через Figma MCP

Перед написанием кода — полный обход дерева через `figma-console`:
- Считать все размеры, отступы, gap, radius, шрифты
- Для каждого цвета: определить токен по роли элемента
- Не угадывать — только реальные значения из узлов
- Подробности: `docs/09_figma_to_code.md`

### 3. Цветовые токены

Все цвета выносить в `tokens/colors.ts`.  
Источник истины — `docs/COLOR_TOKENS.md`.

Если цвет из Figma не покрыт токеном:
1. Не хардкодить
2. Определить роль и предложить имя токена
3. Добавить в `docs/COLOR_TOKENS.md` и `tokens/colors.ts`
4. Сообщить пользователю

### 4. React-компоненты

- Каждый логический блок макета = отдельный компонент
- Данные через props, мок отдельно в `mocks/`
- Иерархия компонентов повторяет иерархию фреймов Figma
- Подробности: `docs/09_figma_to_code.md`

### 5. Правила из репозитория

При вёрстке агент **обязан** учитывать:
- `docs/COLOR_TOKENS.md` — цвета
- `figma/naming_rules.md` — стили, отступы (сетка ×8), шрифт Inter
- `docs/08_constraints.md` — технические ограничения
- `docs/02_design_principles.md` — принципы UI

### 6. Перепроверка

После первой итерации агент **обязан**:
1. Визуально сравнить результат с макетом
2. Проверить размеры, отступы, шрифты
3. Проверить что все цвета через токены
4. Если есть расхождения — исправить и повторить

Расхождение в 1px, неверный токен, неточный gap — это баг. Исправлять.

### 7. Перенос правок из браузерного превью

Когда правки сделаны в DevTools:
1. Идентифицировать элемент по `reactComponent.name` и содержимому
2. Найти в коде — inline-стиль, CSS-модуль или Tailwind
3. Применить изменение в исходнике
4. Не коммитить сгенерированные файлы (`dist/`, `node_modules/`)

---

## Чеклист качества

- [ ] Все значения считаны из Figma, ничего не угадано
- [ ] Цвета только через `colors.*` из токенов
- [ ] Нет hex/rgb в компонентах
- [ ] Компоненты разбиты по логике макета
- [ ] Данные через props, мок в `mocks/`
- [ ] Сетка отступов кратна 8
- [ ] Шрифт — только Inter (400, 500, 600)
- [ ] Результат визуально идентичен макету
- [ ] Перепроверка выполнена, расхождения устранены
