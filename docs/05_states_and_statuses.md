# States & Statuses

> Токены цветов — `docs/COLOR_TOKENS.md`. При вёрстке использовать только их, не хардкодить.

---

## Статусная модель задач

| Статус | Токен текста | Токен фона | Описание |
|--------|-------------|------------|----------|
| Поставлена | `text/secondary` | `bg/elevated` | Задача создана, ожидает принятия |
| Принята | `status/warning` | `status/warningBg` | Задача принята в работу |
| В работе | `primary/default` | `primary/muted` | Активное выполнение |
| Выполнена | `status/success` | `status/successBg` | Задача завершена |
| Просрочена | `status/error` | `status/errorBg` | Дедлайн истёк — визуально давит |

> Точные hex и VariableID для каждого токена — в `docs/COLOR_TOKENS.md`.

---

## Принципы статусов

1. Статусы читаются мгновенно — цвет + форма, не только текст
2. Нельзя путать действия и состояния
3. Просроченные задачи давят визуально — это рабочая система, не мягкий таск-менеджер
4. Цвет — основной носитель информации, текст — уточнение

## UI States

| State | Когда | Визуал |
|-------|-------|--------|
| Default | Обычное состояние | Базовые токены компонента |
| Hover | Курсор над элементом | `primary/hover` для акцентных, `bg/elevated` для нейтральных |
| Active / Selected | Выбранный элемент | `primary/muted` фон + `primary/default` акцент |
| Disabled | Недоступное действие | `text/disabled`, `bg/input`, нет cursor pointer |
| Focus | Фокус с клавиатуры | `primary/default` обводка 2px |
| Error | Ошибка валидации | `status/error` обводка + `status/errorBg` фон |
| Loading | Загрузка данных | Спиннер из иконки `linear/Loading`, `neutral/muted` |

---

## Что критично

- Статусы понятны без чтения текста
- Цвет — основной носитель информации
- Второстепенное приглушено (`text/secondary`), важное выделено (`primary/default`, статусные цвета)
- Однозначность статуса нельзя размывать
- Просроченное — всегда `status/error`, без исключений
