# States & Statuses

## Статусная модель задач

| Статус | Цвет | Описание |
|--------|------|----------|
| Поставлена | Серо-голубой | Задача создана, ожидает принятия |
| Принята | Тёплый оранжевый | Задача принята в работу |
| В работе | Синий | Активное выполнение |
| Выполнена | Зелёный | Задача завершена |

## Принципы статусов

1. Статусы должны читаться мгновенно — цвет + логика > текст
2. Нельзя путать действия и состояния
3. Просроченные задачи должны давить визуально — это рабочая система, не мягкий таск-менеджер

## Что критично

- Статусы понятны без чтения текста
- Цвет — основной носитель информации
- Второстепенное приглушено, важное выделено
- Нельзя размывать однозначность статуса
