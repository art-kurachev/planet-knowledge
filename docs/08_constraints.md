# Constraints

## Технические ограничения

### Шрифт
- **Inter** — единственный допустимый шрифт
- Доступные начертания: Regular (400), Medium (500), Semi Bold (600)
- Начертание 450 используется только во фронтенде (нестандартное для Inter — уточнять у автора)

### Figma API

1. **Цвета — только Variables** — `figma.variables.setBoundVariableForPaint()`, никогда не хардкодить hex
2. **Async API** — использовать `figma.getNodeByIdAsync()`, не `figma.getNodeById()`; `getVariableByIdAsync()`, не `getVariableById()`
3. **После `arrange_component_set`** — ID вариантов меняются; заново читать `set.children` перед применением стилей
4. **Иконки** — стиль ставить только на Vector внутри Instance, не на сам Instance-фрейм
5. **Иконки в табах** — используют `Secondary/BlueGray` / `neutral/muted` в обоих состояниях (active и inactive)
6. **Скриншоты** — не делать для пользователя (он видит макеты в Figma). Использовать только для собственного анализа агента
7. **`primaryAxisSizingMode=AUTO`** — не пересчитывает высоту если фрейм был FIXED. Для changelog и подобных фреймов: `FIXED` + вручную считать и задавать высоту через `resize(w, h)`
8. **`textAutoResize`** — для описаний всегда `WIDTH_AND_HEIGHT`, иначе текст обрежется
9. **`rescaleContent`** — read-only на Instance. Для масштабирования иконок оборачивать во Frame с `CENTER/CENTER`
10. **`figma.currentPage =`** — не работает. Использовать `await figma.setCurrentPageAsync(page)`
11. **Changelog-контейнер** — запись вставляется в конец `304:1445`. Родитель `304:1205` имеет `clipsContent=true` — при добавлении записей проверять суммарную высоту и при необходимости увеличивать вручную
12. **Глубина обхода дерева** — использовать `depth > 15` (не 7). Фреймы в макетах бывают глубокими

## Дизайн-ограничения

- Светлая тема. Dark mode — в разработке (см. `docs/COLOR_TOKENS.md`)
- Основной цвет — синий
- Белые блоки (`bg/surface`) как основные контейнеры
- UI: контрастный, читаемый, структурный, без декоративности
- Рабочая среда 8+ часов/день — визуальный комфорт критичен

## Что нельзя ломать

- Логику статусов
- Последовательность процессов
- Контекст возврата (пользователь должен понимать, куда вернётся)
- Прямоту действий
- Однозначность интерфейса
