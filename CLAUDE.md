# Planet Knowledge — Agent Instructions

Этот файл читается Claude Code автоматически. Для Claude (chat) и Cursor правила тоже здесь.

## Перед началом работы

Прочитай файлы из репо в этом порядке:

1. `docs/02_design_principles.md` — принципы дизайна и инструкции для агента
2. `docs/08_constraints.md` — технические и дизайн-ограничения
3. `figma/naming_rules.md` — правила именования и стили
4. `design-tokens/colors.json` — цветовые токены
5. `figma/file_map.md` — карта Figma-файлов

Остальные файлы (personas, components, flows и др.) — по необходимости.

## При вёрстке фронтенда

Обязательно прочитай `docs/09_frontend_rules.md` — там правила структуры, токенов, синхронизации с Figma.

Ключевые моменты:
- Источник истины — Figma. Не додумывай цвета, отступы, фон.
- Когда стало 1:1 с макетом — остановись.
- Цвета только из `tokens/colors.ts`, не хардкодить.
- Не дублировать компоненты — выноси в `shared/` или `icons/`.
- Моки отдельно от компонентов — в `mocks/`.
- Компонент > 300 строк — разбивай.

## Общие принципы

- Всегда опирайся на данные из репо, а не на устаревший контекст
- Не предлагай геймификацию, «мягкие» UX-решения, скрытые автодействия
- Если решение делает интерфейс сложнее — оно не подходит
