# Planet Knowledge

База знаний о продукте для AI-агентов. Содержит дизайн-принципы, токены, описание компонентов, флоу и решения — всё, что нужно агенту, чтобы давать точные UX/UI советы.

## Структура

```
/docs                        — документация и справочные материалы
  00_product_context.md      — контекст продукта
  01_personas.md             — персоны пользователей
  02_design_principles.md    — принципы дизайна + инструкции для агентов
  03_layout_system.md        — сетка и лейаут
  04_components.md           — каталог компонентов
  05_states_and_statuses.md  — состояния и статусы
  06_flows.md                — пользовательские сценарии
  07_decisions_log.md        — лог дизайн-решений
  08_constraints.md          — ограничения
  09_figma_to_code.md        — технические правила верстки по Figma
  10_coding_workflow.md      — процесс работы агента при вёрстке

/design-tokens               — дизайн-токены в JSON
  colors.json                — цвета

/figma                       — мета-информация о Figma-файлах
  file_map.md                — карта файлов
  naming_rules.md            — правила именования и стили
  component_map.md           — маппинг Figma → Code (синхронизация)
```

## Как использовать

Подключи этот репозиторий как контекст для AI-агента. Агент должен:

1. Читать `docs/02_design_principles.md` перед каждым UX/UI советом
2. Сверяться с `design-tokens/` при работе с цветами, отступами и типографикой
3. Записывать принятые решения в `docs/07_decisions_log.md`

## Синхронизация Figma → Код

Макеты в Figma связаны с React-компонентами через маппинг `figma/component_map.md`.
Каждая нода в Figma привязана к конкретному .tsx файлу по node ID.

### Как запустить синхронизацию

Скажи агенту одну из команд:

| Команда | Что произойдёт |
|---|---|
| **«синхронизируй»** | Проверит все связанные ноды в Figma → найдёт расхождения → обновит .tsx файлы |
| **«проверь Сводку»** | Синхронизация одного конкретного компонента |
| **«что изменилось в фигме?»** | Только отчёт, без правок кода |
| **«синхронизируй принудительно»** | Перезапись по Figma, игнорируя локи |

### Что обновляется, а что нет

**Figma — источник визуала.** При синхронизации агент обновляет только CSS-свойства в styles-объектах: размеры, отступы, цвета, шрифты, borderRadius, gap.

**Код — источник логики.** Агент никогда не трогает: JSX-структуру, props, хуки, обработчики событий, импорты, бизнес-логику. Всё что было поправлено через Cursor или вручную — остаётся на месте.

### Защита ручных правок (Code Locks)

Если ты поправил визуальное значение в коде (например, отступ) и не хочешь чтобы синхронизация его откатила — поставь лок:

```tsx
// Одна строка
padding: 12, // @figma-lock

// Целый блок
// @figma-lock-start
const customHover = { background: colors.primary.muted };
// @figma-lock-end
```

Агент увидит `@figma-lock` и пропустит эту строку/блок при синхронизации.

| Действие | Как |
|---|---|
| Защитить значение | Добавить `// @figma-lock` в конце строки |
| Защитить блок | Обернуть в `// @figma-lock-start` ... `// @figma-lock-end` |
| Снять защиту | Сказать агенту: «сними лок на SummaryCard» |
| Перезаписать всё | Сказать: «синхронизируй принудительно» |
